About

Have you ever been on shopping, found an amazing dress, but struggled finding your size? Now there is a quick solution!
With WearIsIt you can find what you are looking for with a press of a button

How it Works?

Each item has a unique QR-code attached to its tag, containing the model name, size and a unique number.
WearIsIt is consisted of two different mobile applications - one for the customers and the other for the store workers.

  • Easily keep track of items in the store
  • Make your employees work more efficiently
  • Improve your customers experience

Features - Worker App

Secure Authentication

Each worker receives its own username and password.
The password is going through a process of encryption, and saved encrypted in Azure DB.

Add new items

A new item delivered to store can be added to the system by simple QR scanning and choosing an initial location.

Update locations

The worker updates the item locations according their movement in the store

Add Image and tags

Optional feature to connect between an item and its image, and automated process that delievers the most suitable tags.

Features - Customer App

Secure Authentication

The customer can enter as a guest or signup to the app.
His password is saved encrypted in Azure DB.

Find your item

With a simple QR scanning the customer can see sizes and quantities of an item in all locations.

Search by tags

Looking for something specific? Tell us what your item should look like, and we'll find it for you.

Wishlist

Saw something you loved? Put it on your wishlist and come back to it later.

Top Items

Want to know what other customers like the most?
Open the Top items list and get notified when it changes.

Azure Services



Table storage

We used the tables for a number of purposes: storing the credentials of the users in an encrypted manner, managing the inventory and the items locations in the store, and saving tags that characterize the items for a more efficient search in the app.

Blob storage

Assign pictures to the different models.

IOT Hub

IoT hub connects us between our simulated IoT device and Azure cloud services.
By registering a simulated device to our ioT hub we route messages from ioT hub to event hub service.

Event hub

We created the event hub which receives and processes the events sent from the app to update the storages in the cloud.

Function apps: Http trigger and Event trigger

Each reading and updating of information in the cloud was done by calling the function app that accessed the appropriate storage table and read / write to the designated field. We used 2 types of functions: Event Trigger when scanning the barcode of a new item, and Http Trigger for the rest of the functionalities.

SignalR service

This service allows catching changes in real time without the need to refresh the screen. We used this service to inform the customer when an item in which he expressed interest is being moved. The importance of using this service comes from the fact that during business hours, items change their location frequently.

Computer vision - Image processing

Advanced image processing service, which receives an image of an item as an input, and produces information about it, such as: color, fabric type, etc. This information is translated into tags which are used later for an efficient and customized search of items.

Additional - Cryptography

In order to protect both workers’ privacy and registered customers’ privacy, their passwords are saved encrypted in Azure storage, using hash function and salt. The encryption (and decryption) itself occurs at a particular function app, as the workers’ and registered customers’ passwords are sent in an HTTP POST method in order to make login and signup processes more secure.
